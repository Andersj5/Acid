ACID

A Javascript Framework From The Future. Light, Fast, Easy, Scalable.
A framework for creating futuristic applications.

Acid is a modern javascript framework with a focus on current & future APIs (ES7).
Most Acid methods are accessed directly from native objects, however all methods can be pre-fixed.
Status:BETA v5.9 Documentation is currently under construction!

Architecture

Synvigil Architecture

(Syn,SV)

Synvigil is an application architecture written in ECMAScript for web applications. Syn is designed to scale with applications as they grow. Syn is fully modularized, async & built with 60fps environments in-mind.

  • Types of Models
    • Model
      • Standard Model
      • N.O (Not Observed)
      • N.R (Not Reactive)
      • Plain object model
      • Lean option
      • Default without lean has ._ property (["name", "destroy", "save", "restore", "history", "historyClear", "historyRemove", "methods", "clear", "copy"])
    • Plant
      • Generates any type of data structure
      • Can be rendered repeatedly
      • Observable
      • Reactive
      • Private Data Store
    • Factory
      • Manage child components
      • Render child components
      • Public Data Shared across components all notified when changed
      • Subscribe-able (Public & Private subscription types supported)
      • Has object Shares
      • Private Data not shared with child components when changed
      • Observable
      • Reactive
      • Can be rendered repeatedly
      • Supports Templating
      • Node cache -> accessed via nodes object property
      • mountable
      • unmountable
      • killable
      • destroy method that cleans up rendered components as well
      • notifyComponents
      • Get all components
      • Unmount all components
      • Mount all components
      • Kill all components
      • Destroy All components
    • Router (URL & State data routing)
      • They receive change records and then process the path in which to push data through.
      • Easily set options for automatic URL state and history control
      • Supports pushState
    • Base
      • For storing data only
      • Other R.O.Ms (reactive observable models) including bases can subscribe to change events
      • Can sync data from ROMs they are subscribed to
      • Public & Private Data
      • Sub & Notify methods
      • Extra notify methods to reach across many models
  • Modules
    • Cache methods
    • Import models/css/html and other resources on the fly
    • Models are put into the function arguments
    • Compiled for continual use
    • callback supported
    • pre-compilable
    • savable
  • Define
    • Basically a more straight forward module
    • Designed for quick and dirty use
  • Cache system
    • Fast object cache system to share data across the application

  • System is fully configurable

    • Credits for XHR requests (app wide authentication parameters for XHR)
    • App wide analytic support from any internal or external 3rd party library
      • Enabled for all events
      • Enabled for web workers
      • Enabled for XHR requests
